LARGE FILE HANDLING - BATCH PROCESSING STRATEGY
================================================================================

**WHEN TO USE BATCH PROCESSING:**
- Input file contains >100 rows
- Processing time exceeds 10 minutes
- Quality and accuracy need to be maintained

**AUTOMATIC TRANCHING LOGIC:**

File Size       | Recommended Tranche Size | Rationale
----------------|--------------------------|----------------------------------
1-100 rows      | Process all at once      | Single batch optimal
101-500 rows    | 50 rows per tranche      | Balance speed and accuracy
501-1000 rows   | 40 rows per tranche      | Maintain search quality
1001-2000 rows  | 30 rows per tranche      | Prevent rate limiting
2000+ rows      | 25 rows per tranche      | Ensure consistency

**TRANCHE PROCESSING PROTOCOL:**

1. **INITIAL ASSESSMENT**
   - Count total records in input file
   - Calculate number of tranches needed
   - Announce processing plan to user:
   
   Example: "I'll process these 850 records in 22 tranches of 40 records each to maintain accuracy and manage web search rate limits. I'll provide updates after each tranche."

2. **PROCESSING EACH TRANCHE**
   - Process records sequentially within tranche
   - Complete all validations for each record
   - Apply full URL extraction (direct pages only)
   
3. **PROGRESS UPDATES**
   After each tranche, report:
   - Tranche number completed (e.g., "Tranche 5/22 complete")
   - Records processed so far (e.g., "200/850 records validated")
   - Quality metrics: 
     * Confirmed: X records
     * Potential Match: Y records
     * Manual Review: Z records
   - Estimated time remaining (optional)

4. **CONSOLIDATION**
   - After all tranches complete, consolidate into single TXT file
   - Verify all rows present
   - Create final TXT file with proper naming
   - Move to output directory and present to user with downloadable link (never file paths)

CHECK 1: LICENSE ASSESSMENT
================================================================================

DECISION LOGIC
--------------

IF license.license_number__v IS EMPTY OR NULL:
    LICENSE_TASK = "Identify HCP License"
    LICENSE_NUMBER output = "MISSING"
    ACTION: Search NMC/State Medical Council to locate license
    
ELIF license.license_status__v = "Inactive" OR "Expired" OR "Suspended":
    LICENSE_TASK = "Verify License Status"
    ACTION: Confirm current status via council website
    
ELIF license.license_status__v = "Active":
    LICENSE_TASK = "License Present"
    ACTION: Optional web verification if requested

LICENSE SEARCH STRATEGY (When LICENSE_TASK = "Identify HCP License")
--------------------------------------------------------------------

Step 1: Determine likely council based on State

State          | Primary Council | Search URL
---------------|-----------------|----------------------------------

Step 2: Search sequence
  1. NMC Indian Medical Register: Search by name
  2. State Medical Council: Search by name + city
  3. NMC Teaching Faculty (if HCO is medical college): Check faculty listing

Step 3: Record findings in LICENSE_SEARCH_RESULT column
  - If found: "[License#] found in [Council]"
  - If not found: "Not found in NMC/[State]MC as of [date]"

================================================================================
CHECK 2: WEB EVIDENCE VALIDATION
================================================================================

STEP 2A: PARSE HCO NAME
-----------------------

Extract institution and department from hco.corporate_name__v:

Pattern: {Institution} - {City} / Dept of {Department}

Examples:
  Input:  "ESIC Medical College & Hospital - Faridabad / Dept of Pathology"
  Output: Institution = "ESIC Medical College Hospital Faridabad"
          Department = "Pathology"

  Input:  "Dr Uday Reddy Clinic - Hyderabad"
  Output: Institution = "Dr Uday Reddy Clinic"
          Department = None (Clinic type)

STEP 2B: WEB SEARCH STRATEGY
----------------------------

Execute searches in priority order:

Search 1 (Tier 1 Sources):
  "{SOURCE_NAME}" "{Institution}" site:*.gov.in OR site:*.nic.in OR site:*.ac.in

Search 2 (Institution Direct):
  "{SOURCE_NAME}" "{Department}" "{CITY}"

Search 3 (Professional Directories):
  "{SOURCE_NAME}" doctor {SPECIALTY} {STATE}

Search 4 (If above fail - Tier 3/4):
  "{SOURCE_NAME}" {CITY} doctor

**CRITICAL: After performing search, you must navigate to the actual result page (not the search results page) and extract the DIRECT destination URL that displays the HCP's affiliation to the HCO. Never provide Google search URLs.**

STEP 2C: EVIDENCE MATCHING CRITERIA
-----------------------------------

Check           | Method              | Pass Criteria
----------------|---------------------|----------------------------------------
Name Similarity | Jaro-Winkler        | ≥70% match between SOURCE_NAME and web
HCO Match       | String containment  | Institution name on same page as HCP
Specialty Match | Semantic alignment  | Web dept aligns with hcp.specialty_1__v
Degree Match    | Keyword presence    | Qualifications match custom_degree fields

**HCO Affiliation Verification**: The evidence page MUST show the HCP's name alongside the institution name, department, or address to confirm active affiliation.

STEP 2D: CONFIDENCE SCORE CALCULATION
-------------------------------------

Base Score: 50 points

Factor              | Condition                    | Points
--------------------|------------------------------|--------
Name Match          | Jaro-Winkler ≥85%            | +20
Name Match          | Jaro-Winkler 70-84%          | +10
HCO Match           | Exact institution match      | +15
HCO Match           | City/partial match           | +10
Specialty Match     | Department aligns            | +10
Degree Match        | Qualification confirmed      | +5
Source Tier 1       | .gov.in / .nic.in / .ac.in   | +5
Source Tier 2       | Hospital official domain     | +3
Source Tier 4       | Aggregator (Practo, etc.)    | -5
Recency             | Evidence >3 years old        | -10
Recency             | Evidence undated             | -3

Final Score = MIN(100, Base + Sum of Points)

URL RULES - CRITICAL REQUIREMENTS
================================================================================

**NEVER Output These:**
  ❌ Google search URLs (google.com/search?q=...)
  ❌ Bing/Yahoo search result pages
  ❌ URLs with tracking parameters (?utm_source=, &authuser=)
  ❌ Cached/archived pages unless original unavailable
  ❌ Login-required pages
  ❌ PDF viewer URLs (should be direct PDF link)
  ❌ Redirect URLs

**ALWAYS Output These:**
  ✅ Direct destination URLs showing HCP affiliation
  ✅ Clean canonical URLs (without tracking parameters)
  ✅ Publicly accessible pages
  ✅ URLs that open directly to content (not search results)
  ✅ Faculty/staff directory pages showing HCP name + institution
  ✅ Department pages listing HCP as member
  ✅ Hospital "About" or "Our Doctors" pages

**URL Extraction Process:**
  1. Perform web search using search strategies above
  2. Identify the most relevant result that confirms HCP-HCO affiliation
  3. **Click through to that actual page** (do not stop at search results)
  4. Extract the direct URL from the browser address bar
  5. Clean the URL (remove tracking parameters)
  6. Verify the page loads and shows HCP information
  7. Record this cleaned, direct URL in EVIDENCE_URL column

**Example - CORRECT vs INCORRECT:**

❌ WRONG: https://www.google.com/search?q=Dr+Mukta+Pujani+ESIC+Faridabad
✅ CORRECT: https://www.mcfaridabad.com/faculty/pathology/dr-mukta-pujani

❌ WRONG: https://www.google.com/url?q=https://hospital.edu/doctors&sa=...
✅ CORRECT: https://hospital.edu/doctors

----------------------------------

---------------
Indian names may appear reordered:
  "Udaykumar Reddy Chapala" vs "Chapala Udaykumar Reddy"
Use Jaro-Winkler algorithm to handle variations. Score ≥70% = acceptable match.
